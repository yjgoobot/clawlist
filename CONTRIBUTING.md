# Contributing to ClawList

Thank you for your interest in contributing to ClawList! We welcome contributions from the community to help improve this AI Agent era project management software.

## Code of Conduct

Please follow our Code of Conduct in all interactions within the project. Be respectful, inclusive, and constructive in all communications.

## How to Contribute

### Reporting Issues

- Check existing issues before submitting a new one
- Provide a clear and descriptive title
- Include steps to reproduce the problem
- Describe the expected vs. actual behavior

### Feature Requests

- Explain the problem you're trying to solve
- Describe your proposed solution
- Consider the impact on existing functionality

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Add tests if applicable
4. Ensure your code follows the project's style guidelines
5. Submit a pull request with a clear description

## Development Setup

1. Fork and clone the repository
2. Install dependencies with `npm install`
3. Set up your local environment variables
4. Run the development server with `npm run dev`

## Coding Guidelines

- Write clean, maintainable code
- Add comments for complex logic
- Follow the existing code style
- Write meaningful commit messages

## Questions?

If you have any questions about contributing, feel free to open an issue or contact the maintainers.

Thank you for contributing to ClawList!
